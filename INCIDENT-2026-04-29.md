# Incident — 2026-04-29 — bootcheck overreach 500'd prod

## Outcome

- Marketing site (`/`): restored, returns 200.
- Middleware (`MIDDLEWARE_INVOCATION_FAILED`): root-caused and fixed.
- Outstanding: `/leaderboard/[slug]` and `/api/cron/grading-recovery` return 500 (route-handler errors, not middleware — pre-existing, were masked by middleware crash).

## Root cause (3 layered failures)

1. **Module-level throw on missing `SUPABASE_JWT_SECRET`** in `src/middleware.ts:195-197`. Env var was never added to Vercel before the merge. Edge middleware crashed at module init → every request 500.
2. **`bootcheck.ts` REQUIRED list overreached.** Demanded Stripe (3 vars), Upstash (2 vars), `SINCH_XMS_CALLBACK_TOKEN`, `APP_TOKEN_SECRET` — none configured on Vercel. Stripe billing not yet set up (Ken: "down the road"). Even after adding `SUPABASE_JWT_SECRET`, bootcheck threw on the next missing var.
3. **bootcheck dep-resolution check incompatible with Edge runtime.** `require.resolve(name)` throws `ReferenceError` in Edge (no `require`). The `try/catch` interpreted the error as "dep missing" → false-positive throw on `@upstash/redis` and `@upstash/ratelimit` even though they're installed.

### Architectural note for future
- Supabase has migrated this project to ECC P-256 asymmetric JWT signing. Legacy HS256 secret still verifies anon/service_role keys but **not** user session tokens. `src/middleware.ts` `verifyAndExtractClaims` does local HS256 verify of user tokens with `SUPABASE_JWT_SECRET`. If user-session JWTs were issued post-P-256 migration, that local verify will fail and `/dashboard` access will silently redirect to login. Untested in this session. If dashboard auth breaks, switch verify path to JWKS via `jose.createRemoteJWKSet(supabaseJwksUrl)`.
